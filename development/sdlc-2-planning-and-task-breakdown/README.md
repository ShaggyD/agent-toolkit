# SDLC 2: Planning and Task Breakdown

Decompose work into small, verifiable tasks with explicit acceptance criteria. This skill enforces the **Plan** phase of the SDLC — breaking specs into implementable units before any code is written.

## Overview

Good task breakdown is the difference between reliable agent execution and a tangled mess. This skill walks through a five-step planning process: enter read-only plan mode, map the dependency graph, slice vertically (not horizontally), write structured tasks with verification steps, and add checkpoints between phases.

## Use When

- You have a spec and need to break it into implementable units
- A task feels too large or vague to start
- Work needs to be parallelized across multiple agents or sessions
- The implementation order isn't obvious
